# Чек-лист тестирования фильтра дохода

Этот документ описывает проверки формы «Уровень дохода» на странице поиска вакансий [hh.ru](https://hh.ru/search/vacancy).

Обозначения:

- **Авто** — проверку целесообразно выполнять Selenium-тестом.
- **Ручной** — требуется оценка тестировщика.
- **Частично** — основную часть можно автоматизировать, но нужна ручная проверка результата.

## Функциональные проверки

| Проверка | Ожидаемый результат | Способ |
|---|---|---|
| Открытие фильтра «Уровень дохода» | Открывается форма настройки дохода | Авто |
| Поле минимального дохода | Поле доступно для ввода числового значения | Авто |
| Доступные периоды | Отображаются «За месяц», «За смену», «За час», «За вахту», «За услугу» | Авто |
| Минимальный доход | Указанная сумма применяется к поиску | Авто |
| Период дохода | Выбранный период передаётся в параметре `salary_mode` | Авто |
| Флажок «Указан доход» | В поисковом URL появляется `label=with_salary` | Авто |
| Сохранение значений | После применения фильтра сумма, период и флажок сохраняются | Авто |
| Повторное открытие формы | В форме видны ранее выбранные параметры | Авто |
| Очистка суммы | Ограничение по минимальному доходу снимается | Авто |
| Повторное изменение суммы | Результаты поиска обновляются по новому значению | Авто |
| Совместимость фильтров | Фильтр дохода работает вместе с регионом, опытом, графиком и поисковой строкой | Авто, ключевые комбинации |
| Соответствие выдачи | Вакансии в выдаче соответствуют выбранному условию дохода | Частично |

## Негативные сценарии

| Сценарий | Ожидаемый результат | Способ |
|---|---|---|
| Пустая сумма | Фильтр по сумме не применяется; интерфейс не показывает ошибку | Авто |
| Нулевое значение | Значение не применяется либо появляется понятная валидация | Авто |
| Отрицательная сумма | Значение не принимается или отображается ошибка | Авто |
| Буквы и специальные символы | Поле принимает только допустимые числовые символы | Авто |
| Дробная сумма | Значение отклоняется либо нормализуется по правилам сайта | Авто |
| Очень большая сумма | Интерфейс не ломается; значение корректно валидируется | Авто |
| Форматированная сумма | `100 000` и `100,000` корректно обрабатываются либо валидируются | Авто |
| Нет выбранного периода | Используется период по умолчанию либо отображается валидация | Авто |
| Многократное нажатие «Сохранить» | Отправляется один корректный запрос без дублирования и ошибок | Авто |
| Медленный ответ сервера | Страница не зависает; видно состояние загрузки или ошибку | Частично |
| Ошибка сервера | Ошибка обрабатывается без поломки интерфейса | Частично |
| Нет вакансий по условию | Отображается корректное пустое состояние | Авто |
| Изменение фильтра после пустой выдачи | Список вакансий корректно обновляется | Авто |

## Границы автоматизации

В первую очередь автоматизируются проверки состояний формы, валидации ввода и параметров URL: они повторяемы и дают однозначный результат. Для них подходит текущий стек `pytest + Selenium`.

Ручными остаются визуальная оценка, UX, понятность текстов и смысловая проверка релевантности вакансий. Последняя зависит от динамичных данных hh.ru: тест может подтвердить применённый фильтр, но не должен делать ненадёжные выводы о содержании каждой вакансии.
