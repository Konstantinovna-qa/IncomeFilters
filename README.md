# IncomeFilters

Небольшой учебный проект с функциональными тестами Selenium для фильтра «Уровень дохода» на [странице поиска вакансий hh.ru](https://hh.ru/search/vacancy).

Тесты управляют настоящим браузером Chrome, взаимодействуют с пользовательским интерфейсом сайта и проверяют, что выбранные параметры действительно передаются в URL поискового запроса.

## Что проверяется

- открытие формы фильтра дохода;
- доступность периодов выплаты: месяц, смена, час, вахта и услуга;
- применение минимальной суммы дохода;
- включение опции «Указан доход»;
- передачу выбранного периода и суммы в параметры поиска.

## Состав проекта

```text
IncomeFilters/
├── tests/
│   ├── conftest.py                    # Фикстура Chrome WebDriver
│   ├── test_income_filter.py          # Функциональные сценарии pytest
│   └── pages/
│       └── vacancy_search_page.py     # Page Object для страницы hh.ru
├── requirements.txt                   # pytest и Selenium
├── pytest.ini                         # Настройки pytest
└── README.md                          # Эта документация
```

### Page Object

`VacancySearchPage` из `tests/pages/vacancy_search_page.py` изолирует работу с интерфейсом hh.ru от самих тестов. Его методы позволяют открыть страницу и фильтр, задать сумму, период, опцию «Указан доход», применить настройки и прочитать параметры текущего URL.

Для поиска элементов используются `data-qa`-атрибуты. Они менее зависимы от визуальных CSS-классов и делают тесты устойчивее к изменениям оформления сайта.

## Запуск

Требования: Python 3.10+, Google Chrome и доступ в интернет. Selenium Manager автоматически подберёт совместимый ChromeDriver.

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
pytest
```

По умолчанию Chrome запускается без интерфейса. Чтобы наблюдать выполнение сценариев:

```powershell
$env:HH_HEADLESS = "false"
pytest
```

## Особенности

Это E2E-тесты внешнего сайта, поэтому результат зависит от доступности hh.ru и актуальности его интерфейса. При изменении формы фильтра в первую очередь стоит проверить локаторы, собранные в `VacancySearchPage`.
